Overview

How it works

Three steps from raw footage to organized clips — no manual work required.

1

Detect

YOLOv8 nano scans every few frames of your video, flagging every timestamp where a horse appears on screen.

2

Group

Nearby detections are merged into a single "jump event." A configurable gap threshold separates distinct rides.

3

Extract

ffmpeg cuts each event into its own clip with buffer padding before and after — no re-encoding, lightning fast.

Requirements

Prerequisites

You'll need Python 3.9+ and ffmpeg installed before getting started.

Install ffmpeg for your platform:

macOS (Homebrew)

bash 
brew install ffmpeg

Ubuntu / Debian

bash 
sudo apt install ffmpeg
Setup

Installation

Get up and running in under two minutes.

1 Clone the repository
bash
git clone https://github.com/mschiff05/rider-finder.git
cd rider-finder
2 Create a virtual environment
bash
python3 -m venv .venv
3 Activate the virtual environment

macOS / Linux

bash
source .venv/bin/activate

Windows

cmd
.venv\Scripts\activate
4 Install dependencies
bash
pip install -r requirements.txt

💡 On first run, Rider Finder will automatically download the YOLOv8 nano model weights (~6 MB). An internet connection is required the first time only.

Usage

Running Rider Finder

Point it at any video file and let it run. Clips appear in a clips/ folder automatically.

Basic — run on any video
bash
python extract_jump_clips.py path/to/your/video.mp4
Custom output directory
bash
python extract_jump_clips.py video.mp4 --output_dir ./my_clips
Adjust buffer and gap thresholds
bash
python extract_jump_clips.py video.mp4 \
  --output_dir ./clips \
  --buffer 8 \
  --gap 20
Reference

Arguments

All available command-line options.

Argument Default Description
video_file required Path to the input video file
--output_dir <video_dir>/clips/ Directory to save extracted clips
--buffer 5.0 Seconds of padding to add before and after each detected jump event
--gap 15.0 Seconds of no horse detection needed to start a new separate event
Tips & Troubleshooting

Getting the best results

Common adjustments for different footage types.

🔗

Clips getting merged together?

Increase --gap to separate events that are being combined into one clip.

🔍

Missing some jumps?

Decrease --gap for more sensitivity, or increase --buffer to capture more footage around each event.

⚡

Clips cut off too early?

Increase --buffer (e.g. --buffer 10) to add more seconds before and after each jump.

📹

Supported formats

Any format supported by ffmpeg: .mp4, .mov, .avi, .mkv, .mts, and more.
